## Discussion

Contexte de l'entreprise ANTS :

- clients cibles : médecin, avocats, etc... car ils ont besoin d'IA, mais ne peuvent pas forcément utiliser les géants
  en raison des lois/règles de confidentialité de leur domaine (données de santés, très confidentielles)
- depuis ~2021

But du produit :

- fournir une interface avec application IA (Open WebUI), n8n, OpenClaw, ...
- créer une sorte de "PlayStation" de l'ia
- logique de stockage distribué permet de créer la mémoire de l'IA (document, souvenirs, contexte
  entreprise/métier, ...)
- donc lorsque le client ajoute une machine, cela revient à augmenter la capacité de la mémoire "knowledge base"

Retour sur la problématique du "client" pour voir et contrôler le cluster :  
si c'est un programme à installer : pas possible !!  
Rien ne doit être installé, sinon ça casse le concept de base du produit : les utilisateurs pourrait être des
grand-mêres  
Alors tout doit être dans une interface web

Changement de plan : plutôt qu'avoir un programme client dédié, l'interface d'utilisation web et le client sont la meme
chose.  
Il y aura donc un onglet "Settings" à côté des onglets des différentes applications

Cela ramène la problématique de connaitre l'adresse ip d'une / toutes les machines pour l'entrer dans le navigateur.

Option primaire : broadcast un nom de domaine, par exemple `ants.local`  
Option fallback : utiliser l'écran de la machine pour afficher l'adresse ip et/ou un qrcode

Pour l'instant, on considère l'utilisateur connait ip, sinon on sort un peu du périmètre du projet.  
Idéalement, on a un nom de domaine au lieu de l'ip (`ants.local`)
